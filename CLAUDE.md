## Project Overview

This is Parul Sharma's personal portfolio website — a Create React App (React 18 + React Router) whose static case-study pages live in `public/`.

- **Main React App** (`src/`): the homepage / work mosaic. Project cards route to `/enterprise`, `/teams`, etc., which redirect to the matching static HTML page.
- **Static case-study pages** (`public/*.html`): the actual project write-ups (enterprise, healthcare, rasa, teams, murals, windows, template, …), styled with the shared design tokens + `public/css/template.css`.
- **Gatsby Portfolio** (`new-portfolio/`): a separate, unused-in-prod Gatsby experiment ("Jodie" starter).
- **Legacy archives** (`v1/`, `v2/`): old static portfolio versions, kept for reference only — NOT deployed.

## ⚠️ Single source of truth: `public/`

The deploy pipeline (`.github/workflows/deploy.yml`) runs `npm run build` (CRA), which copies **`public/` → `build/`** and deploys `build/` to GitHub Pages via GitHub Actions. Therefore:

- **Edit pages and assets ONLY in `public/`** (`public/*.html`, `public/css/`, `public/img/`, `public/js/`). That is the one place that ships.
- **`build/` is generated output** — gitignored, regenerated by CI on every push. Never edit or commit it.
- There are **no live root-level `*.html` / `css/` / `img/` files** anymore (they were dead duplicates of `public/` that had silently diverged — removed June 2026). Don't recreate them.

```bash
cd new-portfolio
npm run develop    # Start Gatsby development server

```

## Architecture

### Main React App Structure
- `src/App.js`: Main app component with React Router setup + the work mosaic. Project routes (`/enterprise`, `/teams`) are thin components that redirect to the static `public/*.html` page.
- `src/index.js`: Entry point with React 18 root rendering; imports `src/index.css`.
- `public/`: **the deployed web root** — static case-study HTML, plus `css/`, `img/`, `js/` and the shared `css/template.css` + `js/nav-component.js`.

### Gatsby Portfolio Structure
- `config/`: Site configuration and theme settings
- `content/projects/`: Project data in YAML format with associated images
- `src/pages/`: Gatsby page components
- `src/components/`: Reusable React components with TypeScript
- `src/templates/`: Template components for dynamic pages

### Static Assets
- `public/img/`: Image assets organized by project (enterprise, healthcare, rasa, microsoft, etc.)
- `public/unitegallery/`: Third-party gallery plugin for image displays

## Key Features
